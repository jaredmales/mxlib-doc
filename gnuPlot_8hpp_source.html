96 /// An interactive c++ interface to gnuplot
97 /** Spawns a gnuplot sesssion and communicates with it.
98  * \ingroup plotting
99  *
100  * \todo Use mxError for error reporting.
101  *
102  * An example of using gnuPlot to plot data from a file:
103  * \code
104  * gnuPlot gp; //This automatically connects, and is now ready to plot.
105  *
106  * gp.plot("my_data_file.dat", "u 1:2 w l"); //Plots the data in the file
107  * \endcode
108  *
109  * Arrays can be plotted directly:
110  * \code
111  * std::vector<float> x;
112  * std::vector<double> y;
113  *
114  * //...code to populate x and y
115  *
116  * gp.plot(x, y); //Plots y vs x.
117  * \endcode
118  * Note that the vector data types are different -- this is handled automatically.
119  *
120  * To get a response from the gnuplot session:
121  * \code
122  * std::cout << gp.getResponse("show terminal") << "\n"; //Sends the command to gnuplot, gets the response.
123  * \endcode
124  * Error checking and reporting is not straightforward since gnuplot does not return a result if there is no error. If there is an error,
125  * it can take several hundred milliseconds for a response to be available. So for a typical
126  * plot command, say, one does not want to try to read the stderr ouput just in case as this will just time out with a delay.
127  *
128  * For cases where a response is expected, the following example shows how to check for errors.
129  * \code
130  * errno = 0;
131  * std::string response = gp.getResponse("show terminal"); //Waits up to 0.5 seconds, or a user specified timeout.
132  *
133  * if(response == "") //This indicates some error occurred, or a timeout.
134  * {
135  * if(gp.gpError())
136  * {
137  * std::cerr << "gnuplot returned error:\n" << gp.gpErrorMsg() << "\n";
138  * }
139  * else if(errno)
140  * {
141  * perror("error getting response: ");
142  * }
143  * else
144  * {
145  * std::cerr << "timed out\n";
146  * }
147  * }
148  * else
149  * {
150  * std::cout << response << "\n";
151  * }
152  *
153  * \endcode
154  */

233  /** It typically takes some time for a response from gnuplot to become readable,
234  * whether an error occurs or not. It is often 0.3 seconds or more. If there is no error, there will be no response so waiting
235  * for a response to check for errors after every command can be very time consuming. Thus it is a choice whether to check
236  * for errors from gnuplot after every command, and what timeout to use.

241  * If the response begins with \c gnuplot>, then the response is an error. In this case, checkResponse returns -1 and the gpError flag is set. If gpError is not set
242  * but the return value is -1, then some other error occurred (check \c errno).
