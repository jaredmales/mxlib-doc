template<typename _realT, class _derotFunctObj, typename _evCalcT = double>
struct mx::improc::KLIPreduction< _realT, _derotFunctObj, _evCalcT >

An implementation of the Karhunen-Loeve Image Processing (KLIP) algorithm.

KLIP[24] is a principle components analysis (PCA) based technique for PSF estimation.

Template Parameters
_realTis the floating point type in which to do calculations
_derotFunctObjthe ADIobservation derotator class.
_evCalcTthe real type in which to do eigen-decomposition. Should generally be double for stable results.

int mx::improc::KLIPreduction< _realT, _derotFunctObj, _evCalcT >::m_excludeMethod {HCI::excludeNone}

Controls how reference images are excluded, if at all, from the covariance matrix for each target image based on a minimum criterion.

Can have the following values:

  • HCI::excludeNone = no exclusion, all images included [default]
  • HCI::excludePixel = exclude based on pixels of rotation at the inner edge of the region
  • HCI::excludeAngle = exclude based on degrees of rotation at the inner edge of the region
  • HCI::excludeImno = exclude based on number of images

int mx::improc::KLIPreduction< _realT, _derotFunctObj, _evCalcT >::m_includeMethod {HCI::includeAll}

Controls how number of included images is calculated.

The number of included images is calculated after exclusion is complete. Can have the following values:

  • HCI::includeAll = all remaining images are included [default]
  • HCI::includeCorr = the m_includeRefNum of the remaining images which are most correlated with the target are included
  • HCI::includeTime = the m_includeRefNum of the remaining images which are closest in time to the target are included
  • HCI::includeAngle = the m_includeRefNum of the remaining images which are closest in angle to the target are included
  • HCI::includeImno = the m_includeRefNum of the remaining images which are closest in image number to the target are included

◆ m_includeRefNum

template<typename _realT , class _derotFunctObj , typename _evCalcT = double>
int mx::improc::KLIPreduction< _realT, _derotFunctObj, _evCalcT >::m_includeRefNum {0}

Number of reference images to include in the covariance matrix.

If > 0, then at most this many images, determined by highest cross-correlation, are included. This is determined after rotational/image-number exclusion. If == 0, then all reference images are included.

Specify how the data are centered for PCA within each search region.

Can have the following values:

  • HCI::imageMean = the mean of each image (within the search region) is subtracted from itself
  • HCI::imageMedian = the median of each image (within the search region) is subtracted from itself
  • HCI::imageMode = the mode of each image (within the search region) is subtracted from itself
  • HCI::meanImage = the mean image of the data is subtracted from each image
  • HCI::medianImage = the median image of the data is subtracted from each image

std::vector<int> mx::improc::KLIPreduction< _realT, _derotFunctObj, _evCalcT >::m_Nmodes

Specifies the number of modes to include in the PSF.

The output image is a cube with a plane for each entry in m_Nmodes. Only the number of eigenvalues required for the maximum value of m_Nmodes are calculated, so this can have an important impact on speed.

Can be initialized as:

red.m_Nmodes={5,10,15,20};
